To jest dokumentacja do gry Most(l)y Done. Zawarte tu są informacje dotyczące pobierania programu z repozytorium, użytych algorytmów oraz błedy które napotkaliśmy w czasie trwania projektu. Na dokumentacje do gry składa się ten plik, obrazy w folderze doc a także komentarze zawarte w plikach kodu.

Jak to się zaczeło

Zaczęło się niewinnie – chcieliśmy stworzyć grę o budowaniu mostów, ale zamiast tego zbudowaliśmy... mnóstwo bugów. 😅 Jednak w miarę postępu prac udało nam się stworzyć coś naprawdę fajnego! Most(l)y Done to gra, w której testujesz swoją inżynieryjną kreatywność, projektując mosty zdolne do utrzymania przejeżdżających pojazdów. Brzmi prosto? Poczekaj, aż zobaczysz poziomy z ograniczonym budżetem...

Krótki opis gry

Most(l)y Done to gra logiczno-fizyczna inspirowana klasyką gatunku – Poly Bridge. Twoim zadaniem jest budowa mostów, które muszą wytrzymać przejazd pojazdów na drugą stronę. Brzmi prosto? Tylko do momentu, aż skończy się budżet lub Twój most zawali się pod własnym ciężarem!

Kluczowe mechaniki:

  • Tworzenie konstrukcji z różnych materiałów (stal, drewno, droga)
  • Realistyczna fizyka mostów – źle zaprojektowane mosty się zawalają
  • Ograniczony budżet – każda belka się liczy!
Czy uda Ci się stworzyć idealny most, czy raczej spektakularnie zawalisz całą konstrukcję? 😅 Sprawdź swoje umiejętności inżynieryjne w Most(l)y Done! 🚧

Ograniczenie długości belek między punktami mostu

Funkcja solveDistanceConstraint w physics.js sprawia, że dwa dane punkty oddalone są od siebie o preferowaną długość. Funkcja oblicza odległość pomiedzy połączonymi punktami, a następnie przemieszcza je o wektor obliczany na podstawie stosunku ich mas.

Obraz przedstawiający obliczenia które wykonuje funkcja, a także model przemieszczenia punktów.


Link do filmu tłumaczącego
Przykład zastosowania:

javascript
for (let i = 0; i < bridgeEdges.length; i++) {
    let e = bridgeEdges[i]
    let extent = solveDistanceConstraint(e[0], e[1], e[3], e[2])
}

Napotkane błędy

Sklonowanie samochodu

Podczas tworzenia funkcji do rysowania samochodu oraz symulacji jego grawitacji było potrzebne obrócenie samochodu aby rysunek poprawnie się wyświetlał, podczas próbowania różnych rozwiązań udało się osiągnąć sytuację przedstawioną na obrazku



Jeszcze ciekawsze było ich zachowanie albowiem odpychały się, dolny leciał na dół, a górny w górę. Po zmianach w funkcji gravitacji nie udało się powtórzyć tego błędu

Podczas tego samego testowania z funkcją rysowania samochodu udało się osiągnąć ciekawy efekt.



Napotkaliśmy również na kolejny problem z rysowaniem samochodu, mianowice samochód nie chciał się rysować po odświeżeniu strony więc postanowiliśmy funkcje odpowiedzialną za rysowanie samochodu wywoływać dwa razy

javascript
drawCar()
drawCar() // nie potrzebne dwukrotne wywołanie


Jak to zostało ostatecznie zaimplementowane

javascript
case "shape": {
    switch (physicalObject[3]) {
        case "polygon": {
            setCanvasFillColor(physicalObject[1])
            drawPolygon(physicalObject[6])
            ctx.fill()

            break
        }

        default: {break}
    }

    break
}

Obraca się? Obraca

Podczas implementowania funkcji odpowiadającej za obrót pojazdu, obiekt który symulował samochód zaczął się niekontrolowanie obracać i odbijać od podłoża.



Błąd powstał w wyniku testowania czy funkcja obracania działa.
